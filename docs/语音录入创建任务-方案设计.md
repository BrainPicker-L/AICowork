# 语音录入创建任务功能 - 方案设计（Typeless 式）

## 1. 功能概述

**目标**：用户按住 Fn 键进行语音录入 → 调用语音大模型完成「转写 + 文本优化」→ 用优化后的文本创建新对话并自动启动任务 → 在页面用一个小图标展示该任务执行情况。

**参考**：Typeless 的「语音输入 → 自动成文并执行」的交互形态。

---

## 2. 设置项（新增）

在 **设置** 中新增「语音」或「语音输入」区块，与现有 API 等设置并列，包含以下三项配置。

### 2.1 语音大模型配置

- **定位**：与现有「API 配置」类似，独立的一套接口配置，专用于语音转写及文本优化（端到端由同一套配置完成）。
- **配置项**：
  - **接口类型**：下拉选择（如 Whisper 兼容、阿里百炼 Qwen3-ASR 等）。**产品约定**：下拉列表中「阿里百炼 Qwen3-ASR」排在「Whisper 兼容」之上（百炼优先展示）。
  - **Base URL**：语音 API 的 base URL（如 `https://api.openai.com/v1` 或厂商语音端点）。
  - **API Key / Auth Token**：认证密钥。
  - **模型**（可选）：若该 API 支持多模型，则提供模型选择；否则可隐藏或使用固定模型。
- **测试连接**：配置填写后提供**「测试连接」**按钮；点击后校验当前 Base URL、API Key、接口类型是否可用，并展示测试结果（成功/失败、详情、响应时间），交互与设置 - API 设置 - 添加配置 中的测试链接一致。
- **存储**：与现有 API 配置一致，持久化到本地（如 config store），首版可先做单条「当前使用的语音配置」。
- **使用处**：流程中的「语音 → 文字」及「文本优化」均使用该配置调用；具体 API 形态由用户在设置中接入，实现方不绑定具体厂商。

### 2.2 语音任务默认工作目录（cwd）

- **类型**：单行输入 + 浏览选择（与现有 StartSessionModal 的工作目录选择方式一致）。
- **必填**：**必须配置**。未配置时不允许启动语音流程（即：不响应 Fn 或提示用户先配置工作目录）。
- **使用处**：创建新会话时，将此处配置的 cwd 作为 `session.start` 的 `cwd` 传入。

### 2.3 是否开启 Fn 键语音转写

- **类型**：布尔开关（如「使用 Fn 键进行语音转写」）。
- **默认**：**关闭**，避免未配置或不需要该功能的用户被误触发。
- **行为**：
  - **开启**：在满足前置条件时（见下）监听 Fn 按下/松开，触发录音与后续流水线。
  - **关闭**：不注册 Fn 监听，不响应 Fn 语音录入。
- **前置条件**：仅当「已开启本开关」且「已配置语音大模型」且「已配置语音任务默认工作目录」时，才响应 Fn；否则不触发或提示用户完成设置。

                                    │

│  - 检测 Fn 按下 → 开始录音（麦克风）                                      │

└─────────────────────────────────────────────────────────────────────────┘

│  阶段 3：创建新对话并启动任务                                             │
│  - 用优化后的文本作为 prompt                                              │
│  - cwd：使用设置中的「语音任务默认工作目录」                              │
│  - 生成 session title（可复用现有 generateSessionTitle）                 │
│  - 发送 session.start，与现有逻辑一致                                     │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼

│  阶段 4：任务执行状态展示                                                 │

**错误与边界（已定）**：

- **转写或优化失败**：可将已转写的原始文本填入输入框，供用户编辑后手动发起会话。
- **流程未完成时关闭窗口或退出应用**：取消未完成请求，不创建会话。

## 4. 模块划分（设计层面）

| 模块 | 职责 | 说明 |
|------|------|------|
| **设置：语音大模型** | URL、Key、可选模型的配置与持久化 | 与现有 API 设置类似 |
| **设置：语音任务 cwd** | 语音创建会话时的默认工作目录（必填） | 未配置则不允许启动语音流程 |
| **设置：Fn 开关** | 是否启用 Fn 键触发语音转写 | 默认关闭；与上述两项共同构成前置条件 |
| **快捷键 / 按键监听** | 使用 **Fn**，仅「按住说话、松开结束」 | 仅当三项设置均满足时生效；macOS 上建议仅应用聚焦时监听以避免系统冲突 |
| **录音与音频** | 采集麦克风、输出指定格式 | 格式/采样率需适配所选语音 API |
| **语音转写与优化** | 使用「语音大模型配置」端到端完成转写+优化 | 优化包含分段、润色；具体 API 由用户自行在设置中接入 |
| **任务创建** | 用最终文本 + 设置中的 cwd + title 调用 session.start | 复用现有 IPC 与 handleSessionStart |
| **状态展示** | 小图标 + 四阶段状态，点击跳转本次对话 | 固定窗口一角（如右下角）；状态：录音中 → 转写中（含优化）→ 任务运行中 → 完成/失败 |

## 5. 与现有架构的衔接

- **设置页**：在设置中新增「语音」或「语音输入」区块（可单独导航项），包含：① 语音大模型配置（URL、Key、可选模型）；② 语音任务默认工作目录（必填）；③ 「使用 Fn 键进行语音转写」开关。配置存储复用现有 config/store 模式。
- **会话创建**：完全复用 `session.start` 与 `handleSessionStart`，不新增会话类型。
- **IPC**：可新增事件，例如：客户端 → 主进程 `voice-task.start` / `voice-task.cancel`；主进程 → 客户端 `voice-task.status`（阶段、错误、最终 sessionId 等）。
- **状态展示**：增加「当前语音任务」的轻量状态（如 store 中的 `voiceTaskStatus`），由小图标组件消费；点击跳转至对应会话（本次对话）。

| 语音大模型配置形式 | 与当前设置的 **API 配置/API 接入方式一致**，用户自己填写（Base URL、Key 等），不绑定具体厂商 |
| 语音设置-接口类型 | **接口类型**下拉中「阿里百炼 Qwen3-ASR」排在「Whisper 兼容」之上 |

| 状态展示 | 四阶段：**录音中 → 转写中（含优化）→ 任务运行中 → 完成/失败**；**不显示简短文案** |

| 完成状态气泡 | 完成时展示回复预览气泡；气泡右上角可**关闭**，关闭后悬浮球恢复初始状态（语音(fn)） |

### 6.1 悬浮球点击行为

- **任何时候点击悬浮球（或气泡）**：都会唤起主窗口并进入应用页面。
- **当球为「运行中」或「完成」状态时**：除上述行为外，会跳转到**本次语音任务对应的会话**（即当前/最近这条任务所在会话）。
- 其他状态（空闲、录音中、转写中、转写失败、失败）：仅唤起主窗口，不强制跳转会话。

### 6.2 并行任务时的展示（结论）

- **状态聚合**：球只显示一个主状态，按优先级取「最前」阶段：录音中 &gt; 转写中 &gt; 运行中 &gt; 完成/失败。不展示数量（如「运行中(2)」），首版仅展示单一状态文案。
- **气泡**：仍只展示**最近完成**的那条任务的最后一次大模型回复；点击跳转该条会话。
- **点击**：行为同上（任意点击进应用；运行中/完成时进本次任务会话）。「本次任务」在并行场景下指当前聚合状态所代表的那条（或最近完成的那条，由实现用 sessionId 跟踪）。

### 6.3 完成状态气泡与关闭

- **完成状态**：当悬浮球为「完成」状态时，会展示**气泡**，内容为最近一次大模型回复的简短预览（约 120 字内）。
- **气泡关闭**：气泡右上角提供**关闭按钮**（×）；用户点击后，气泡收起，悬浮球恢复为**初始状态**（「语音(fn)」），不再展示该条任务的会话跳转入口，直至下一次语音任务完成。

## 7. 实现时由开发方定的细节（无产品侧待决项）

以下为纯实现细节，无需再与产品确认：

- 语音大模型配置的 **存储 schema 与默认值**（与现有 API 配置对齐即可）。
- `voice-task.*` **IPC 事件名与 payload** 的 TypeScript 类型定义。
- 小图标组件的 **具体布局与状态枚举**（与文档中四阶段一致）。
- 为支持「运行中点击跳转本次会话」，主进程在会话创建后需向悬浮窗下发带 `sessionId` 的状态（如 `running`），以便球在运行中阶段也能解析出要跳转的 sessionId。
- **长任务时语音无法录入/无法识别**：根因是 Fn 键事件与 `server-event` 共用同一 IPC 通道，长任务产生大量流式事件导致 Fn 的 down/up 在队列中被堵住。修复方式：主进程通过独立通道 `voice-fn-key` 下发 Fn 键，渲染进程用 `registerVoiceFnKey` 订阅，不与 `server-event` 争用。
- **应用关闭**：用户点击主窗口关闭（如左上角关闭按钮）时，应同时销毁悬浮窗并正常退出应用，使任务栏/程序坞图标消失；避免主窗口关闭后悬浮球与图标仍存在、功能不可用的情况。实现上在主窗口的 `close` 事件中调用 `destroyFloatingWindow()`，从而触发「所有窗口已关闭」并执行应用退出流程。

**本方案已包含设置中的三项**：① **语音大模型配置**（与现有 API 设置一致，用户自填 URL、Key 等）；② **语音任务默认工作目录**（必填）；③ **是否开启 Fn 键进行语音转写**。产品侧结论已全部确定，可直接进入实现阶段。
